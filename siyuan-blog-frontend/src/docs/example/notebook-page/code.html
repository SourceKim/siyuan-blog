Database Schema for a Minimalist Blog

Database Siyuan Tech
Sophia Chen
July 26, 2024

Introduction

When designing a backend for a personal blog, especially one with a minimalist aesthetic, the database schema is a critical foundation. This article outlines a simple yet effective schema for a Siyuan-powered blog, focusing on the core entities: Notebooks, Articles, and Tags.

Core Entities

The database will revolve around three main tables. We'll use SQL syntax for clarity. 

-- Notebooks Table: Represents the folder structure
CREATE TABLE Notebooks (
    id VARCHAR(255) PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    parent_id VARCHAR(255),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (parent_id) REFERENCES Notebooks(id)
);
-- Articles Table: Stores the content of each post
CREATE TABLE Articles (
    id VARCHAR(255) PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    content TEXT,
    notebook_id VARCHAR(255),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (notebook_id) REFERENCES Notebooks(id)
);

Schema Explanation

The `Notebooks` table uses a self-referencing `parent_id` to create a hierarchical tree structure, mirroring the directory system in Siyuan. The `Articles` table holds the main content, linked to a specific notebook. This setup allows for easy navigation and organization, which is essential for a clean, minimalist blog experience.
